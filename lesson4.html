Exercise 4: Searching for Files and Words

In most industry settings, projects are collaborative and large-scale. Code bases are tens of thousands to millions of lines in length. In any large-scale project, being able to successfully navigate Linux thus depends on being able to find what you need in the large sea of files. Here, we introduce basic techniques for finding files and content in the Linux file system.

First, let’s assume that you know a phrase located inside some files in your directory, like “cats”. You want to be able to find all instances of the word “cats” in the files inside this folder. To do so, let’s make some files quickly using the skills you learned earlier:

echo “I love cats” > file1.txt
echo “Dogs are cool” > file2.txt
echo “Cats are the best” > file3.txt
echo “Kittens are small” > file4.txt
echo “I want a cat” > file5.txt

Now, let’s try to find all of the matches of the word “cats”. The command grep does exactly this. Let’s first search for it in just one of the files, like file 1. We specify the string we are searching for and then the name of the file:

grep cats file1.txt

You should see the line of the file containing “cats” returned, with the word “cats” highlighted. Now, let’s find where it is in all files by substituting the wildcard character * for the file name:

grep cats *

Now, you can see the name of each file where the word “cats” was found and the corresponding matching line. Finally, let’s try searching for a longer phrase. Now, we need to put it in quotes:

grep “a cat” *

It should be found in file4.txt only.

Next, we want to be able to search for file names as well. To set up, create a directory named simulations in your current directory, move to that directory, and create a file named nov_5_tests.txt containing the phrase “Observation 1: users could navigate the document successfully”. You should know how to create it by now.

Now, move back up a level in the directory. The find command lets us search for files by name. Let’s search for files named “nov_5_tests.txt” in case we forgot where we put it:

find . -name “nov_5_tests.txt”

The dot tells us to start searching in the current directory. It will search recursively, so even though our file is not found by listing the files in this directory, it is in our part of the file “tree” and will be found. You should see the following path to the file: ./testing/nov_5_tests.txt
