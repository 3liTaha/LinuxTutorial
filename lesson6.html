Exercise 6: Process Management

Process management is a useful skill to master when learning Linux. It allows you to pry into the roots of what your system is doing thread by thread. A process in Linux is just what they call any program that is being executed. There are two types of processes Foreground and Background processes. Foreground processes are initiated by the user while Background processes are initiated by the system. Let’s test a Foreground process by typing:

sleep 10

You may have noticed that after running that command your console became inactive. That’s exactly what the sleep command does, it pauses the execution of the next shell command. Linux lets us see all processes that are currently running by typing:

top

This command displays a list of currently running processes along with their CPU and memory usage. The table at first glance can feel very overwhelming and takes time to adjust to. To help you better understand what is going on we have provided you with the list of all the table headings definitions below.

PID: Unique Process ID given to each process.
PPID: stands for "parent process ID", which is the process that created the process
STAT: State in which the process is in.
  D Uninterruptible sleep (usually IO)
  R Running or runnable (on run queue)
  S Interruptible sleep (waiting for an event to complete)
  T Stopped, by a job control signal or because it is being traced.
  X dead (should never be seen)
  Z Defunct ("zombie") process, terminated but not reaped
Additional but not required:
  < high-priority
  N low-priority
  L has pages locked into memory
  s is a session leader
  l is multi-threaded
  + is in the foreground process group
User: Username of the process owner.
VSZ: is short for Virtual Memory Size. It's the total amount of memory a process may hypothetically access.
%VSZ: Percentage of Virtual Memory Size used by the process.
%CPU: Percentage of CPU used by the process.
Command: Command used to activate the process.

Now that we have seen all threads running let’s exit the list by typing q. Now let's say what happens if you want to kill a process. To do this all we need is the PID of the process and the command kill [PID]. Try it out for yourself, but do not wait for a response as kill does not have a response. Processes offer us tons of information but it takes some time to get used to so do not worry if you feel overwhelmed now.’
