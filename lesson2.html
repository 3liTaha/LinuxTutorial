Exercise 2: Linux File System

Being able to manipulate files and their location using Linux commands bolsters your efficiency while working instead of having to utilize the mouse for all those actions. In this exercise we will go over some commonly used commands.

You could create a file in two main ways. If you’re looking to create an empty file, test1.txt you can use:

touch test1.txt

You could also create a file with some text in it using the echo command introduced in the earlier exercise:

echo “let’s start with exercise2” > test1.txt

This command would create the file test1.txt with the text ‘let’s start with exercise2’. Having learned how to create a file, let’s delve into the manipulation part of it. The mv command would be a powerful tool throughout this. If we want to rename test1.txt to test2.txt, we would simply type in

mv test1.txt test2.txt

The mv command also enables us to move the file to another directory. We’ll use the directory created in the previous exercise baby_lions for directory-related commands in this exercise. The following command

mv test2.txt baby_lions

moves the file test2.txt to the directory baby_lions. The last step of file manipulation is knowing how to delete a file. To delete the file test2.txt, the following command suffices:

rm test2.txt

You could delete multiple files by listing all of them in front of rm. Bonus: you can also delete directories using the -d flag. This would translate into:

rm -d baby_lions

We may have lied about being finished. Just one small yet powerful command remains, cat. Don’t be deceived by the name, it’s one of the most powerful commands to have under your use. You can use it to view a file by simply writing:

cat test2.txt

You can copy the contents from one file to another:

cat test1.txt > test2.txt

This copies the contents from test1 to test2. You can also use it to create a new file:

cat > test1.txt

While this may be a lot of information, all of these commands can be associated with their function. For example remove and rm!
