Lesson 3: File Compression and Decompression

The term compressed files has become increasingly common as such files require less storage space and less resources to transfer. To achieve this, Linux once again comes to the help. It’s a complete circle here where you can compress and then go a step further and decompress the file! We first will have to create a file, which has been covered in the previous exercise. Let’s start off with the most simple type of compression:

gzip test1.txt

You guessed it right, this command creates a compressed version of test1.txt and is called test1.gz. The following command would just undo the command above:

gunzip test1.gz

Bonus feature included with linux entails password protecting your compressed file using the -p flag. The command would look like:

gzip -p test1.txt

Now once you try unzipping the password protected file, you will have to enter a password to ensure the process is carried out.

However, Linux also supports various types of compressions. The two main ones being gzip (which has already been introduced) and bzip2. While gzip provides faster compression, the latter ensures a higher compression ratio. As you guessed, the following two commands are responsible for the compression of files:

gzip test1.txt

bzip2 test1.txt

While the following commands would decompress the files:

gunzip test1.gz

bunzip2 test1.bz2

While gzip and gunzip are sufficient for most scenario, we hoped to give a context into other forms of compression and decompression commands. More ways to impress your manager!
